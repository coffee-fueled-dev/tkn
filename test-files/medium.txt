This is a medium-sized test file for demonstrating the TKN file streaming functionality.
It contains multiple paragraphs and repetitive content to create a file that's large enough
to see progress updates during streaming, but small enough to complete quickly.

=== Section 1: Protocol Description ===

The TKN protocol is a simple binary protocol with the following format:

+------+----------------+-----------------------+
| Type | Length (4byte) | Payload (variable)    |
+------+----------------+-----------------------+
   1B        4B             Length bytes

Message Types:
- 1: JSON data
- 2: String data
- 3: Binary data

The length field is a 4-byte big-endian integer that specifies the length of the payload in bytes.

=== Section 3: Streaming Process ===

When streaming this file:
1. The file is opened as a read stream
2. It's piped through a transform that adds delays between chunks
3. Each chunk is sent to the TKN server using the client library
4. Progress is tracked and displayed periodically
5. Upon completion, statistics are shown (bytes sent, time, rate)
